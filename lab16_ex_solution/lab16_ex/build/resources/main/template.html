Lab 10: Tasks for agent $username

All IT systems of MI-6 have been compromised. Her Majesty's agents are at risk.

Your task is to develop a new secret agent management system for MI-6. You have less than 2h to complete your work.

Don't work with anyone else on this. Don't share any information. You may only trust some of the TAs.

Task 1: Configuration

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes

Create class com.example.lab10_ex.domain.Agent. With the following private fields:

  1. int id
  2. String name
  3. String country

Generate necessary getters and setters.

Task 3: Controller (simple)

Create controller com.example.lab10_ex.controller.MainController.

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists).

Task 4: Controller, View, and Form

Create field public static List<Agent> agents in class com.example.lab10_ex.Lab10Application. Initialize the list to contain two agents.

Create controller com.example.lab10_ex.controller.AgentController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/agents" should show view agents/list.
    • Add the list of all agents from com.example.lab10_ex.Lab10Application to the model as attribute agents.
    • Create the corresponding JSP to list the name and country of each agent followed by a link to URL "/terminate?agent={id}", e.g., "/terminate?agent=3".
  2. Requests for URL "/newAgent" should show view agents/form. Create the corresponding JSP with a form with modelAttribute="agent" and action="/addAgent" and inputs for all fields of class Agent.
  3. POST requests for URL "/addAgent" should add a new agent to the list of all agents in com.example.lab10_ex.Lab10Application and redirect to /agents.

Task 5: GET Requests

Extend controller com.example.lab10_ex.controller.AgentController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/terminate" with request parameter agent={id} should delete the agent with the corresponding ID from the list of all agents in com.example.lab10_ex.Lab10Application and redirect to /agents.

Task 6: Validation

Set up a validator for class Agent and make it available in controller AgentController.

When adding a new agent validate that

  1. The value of field id is unique within the list Lab10Application.agents.
  2. The value of field name is not empty.
  3. The value of field country is different from "UK" (MI-6 deploy agents only in foreign countries).
In case of validation errors go back to the form and show error messages for each field in the form.
